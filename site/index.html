Kaisola Mesh

Multiple models. One controlled implementation.

Start with Claude and Codex, add more participants, and select each provider's live model. A Mesh gives the team a shared mission, but not a shared checkout: Kaisola coordinates the handoffs and keeps state-changing stages behind your approval.

Agents without the terminal tax

Rich agent sessions, backed by real CLIs.

Codex and Claude connect through the Agent Client Protocol, giving Kaisola structured messages, tool activity, plans, permission requests, queues, and resumable sessions.

ACP

Codex and Claude, first-class

Read agent output as a conversation while retaining access to the underlying CLI session and its workspace context.

Terminal

A real PTY when you need it

Interactive shells and programs run through node-pty. Watch commands as they run or open a terminal of your own beside the agent.

Queue

Keep the next instruction close

Follow-up messages sit above the composer while an agent is busy. Steer supported sessions mid-turn, or let the queue deliver next.

MCP

One tool surface across agents

Project and user MCP servers can travel with supported agent sessions, while the Kaisola MCP bridge exposes workspace context and gated proposals.

Extensions

Grow the editor deliberately

Install language, grammar, preview, and MCP contributions from Settings. Bundled extensions cover common code, data, document, and media formats.

Control

Permission modes stay visible

Choose how much an agent may do per session. Requests and tool activity remain part of the conversation instead of disappearing behind a spinner.

Local by default

Your workspace does not depend on a Kaisola model cloud.

Kaisola coordinates software installed on your Mac. Your project and working state remain local, and signing into a Kaisola profile is optional.

01

Use your existing agent accounts

Codex and Claude authenticate through their own CLIs. Kaisola does not proxy those agent prompts through a separate Kaisola model service.

02

Continue locally

Google sign-in is optional. You can open projects, edit files, run terminals, and use configured agents without creating a Kaisola profile.

03

OS-backed credential storage

If you do use Google sign-in, its durable refresh credential is encrypted in the main process with Electron safeStorage and is not exposed to the renderer.
