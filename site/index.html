Kaisola is a research IDE that works like a browser for coding agents — claude, codex, gemini, any CLI — running as you, side by side on glass. Group them, pin them, ⌘1–9 between them, and review every change before it lands.

Your agents already live in terminals. Give them a browser.

Chrome made pages tabs. Kaisola makes agent sessions tabs — with the same muscle memory.

Sessions are tabs

Every agent, terminal, git panel, and embedded browser is a card you can pin, group, split, and cycle with ⌘1–9 — pinned tabs, tab groups, and ⌘⇧T reopen included.

Projects are tabs

A Chrome-style strip across the very top holds your repos. Switch projects like switching sites — agents in background projects keep working, and a badge tells you when one needs you.

Yours, on glass

Bring your own claude, codex, opencode — Kaisola never proxies a model. Logins run in real TTYs; keys live in the macOS keychain; the shell is native liquid glass.

A real IDE underneath.

Not a wrapper around a chat box — terminals, git, files, and builds are first-class.

Terminals

Real ptys, not simulations

node-pty terminals with WebGL rendering: the shell draws its own prompt, colors and interactive programs just work. When an agent runs a command, its terminal appears beside it — watch, or take over.

❯ claude # boots signed-in, in your workspace
❯ git worktree add ../fix-123 # isolated agent sandboxes
Agent registry

Ten CLIs, one + menu

Built-in presets for the agents you already pay for — each runs the official CLI with your existing install and auth. Add anything else as a custom entry, ACP or terminal.

Git

Commit beside the agent that made the change

A commit panel card: browse changes, stage hunks, side-by-side diffs, commit — without leaving the window. Checkpoints snapshot the working tree before every agent turn, so any turn is one click to rewind.

Safety

Permission cards, not YOLO mode

Agent asks are inline cards with allow-rules you control; sensitive-file globs are enforced below the agent — it can't read what you've fenced off. Autonomy is a ladder (Observe → Propose → Execute), not a switch.

Writing

LaTeX mode

Open a .tex workspace and the shell leans into paper-writing: latexmk builds, PDF preview with SyncTeX jumps, and Overleaf sync.

Keyboard-first

The whole shell on home row

⌘L talk to any agent · ⌘K palette · ⌘P files · ⌘J cards · ⌘1–9 sessions · ⌃Tab cycle — every chord rebindable from keymap.json.

Built for research that has to hold up.

Underneath the shell is a research IDE. The primitive isn't the paper — it's the trajectory: corpus to claims to hypotheses to experiments to manuscript. Every claim links to its evidence — a citation, a result, a derivation — and carries a computed trust level. Agents never mutate your work: they file proposals, rendered as research diffs, and you approve every transition.
