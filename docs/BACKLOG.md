\- **Port Kaisola from Electron to Swift-native.** Goals: native feel + kill Electron memory (measured ~880 MB/window today), and share one Swift core across Mac + iPhone. [Full design spec](superpowers/specs/2026-07-20-kaisola-swift-native-design.md) (rev3, two rounds of repo-grounded plan-review) · [Phase 0/1 design](superpowers/specs/2026-07-21-kaisola-swift-phase-0-1-design.md) · [implementation plan](superpowers/plans/2026-07-21-kaisola-swift-phase-0-1.md). — **strategy and Phase 0/1 approved; Phase 0 is integrated and the Phase 1 observe-only native preview has crossed the local implementation boundary as of 2026-07-21. It now has the complete read inventory, private cursor checkpoints, bounded reconnect/wake recovery, exact suffix resume, a checksum-pinned universal Node/node-pty helper, sealed per-user bootstrap, independent N/N+1 versions, Sparkle preview-channel integration, synthetic N → N+1 → rollback continuity over one real PTY, a same-metric physical-footprint harness, and automated SwiftTerm framing/scrollback/ANSI/Unicode/resize stress gates. An exact sealed universal `LocalRelease` artifact also passed its native launch probe and a direct AppKit/accessibility-tree shell inspection, including the correct read-only/offline coexistence response to an older live Electron broker and sidebar hide/show. Phase 1 is not complete: paired Electron/native footprint results, the remaining packaged GUI/VoiceOver interaction rows, a real signed Sparkle update with real Claude and Codex processes, Developer ID signing, notarization/stapling, Gatekeeper/translocation, and clean-user/login testing remain open.**

**Strategy — hybrid strangler → fully native.** Never a big-bang rewrite. The Electron app stays the daily driver and keeps shipping until the native app is demonstrably better. A native macOS shell (AppKit/SwiftUI on a shared `KaisolaCore` package) grows by *strangling* — it drives the existing battle-tested backends first, then ports each to Swift one at a time. Editor/docs live in a WKWebView (reusing today's React) at first, native later. The one truly durable, socket-speaking backend today is the detached `session-broker` (owns the PTYs); the native app reuses it unchanged first, then replaces it with a Swift PTY daemon last.

**Headline invariant — durable terminal/agent CLI runs survive app update/restart.** An in-flight `claude`/`codex` run keeps running while you update Kaisola; the relaunched app reattaches with continuous scrollback. Bounded honestly: survives app quit/relaunch/update, but *not* broker crash / logout / reboot, and history is retained up to the ~16 MiB spool with an explicit truncation marker.

**Migration phases (each ships; Electron stays the daily driver until parity):** 0) extract `KaisolaCore`, re-base the iPhone app on it · 1) native SwiftTerm terminal spike (observe-only, no ownership theft) · 2) control-plane host + native ACP / windows · 3) editor/docs via WKWebView bridge · 4) daily-driver parity (auth, settings, provider, MCP, companion, updates) · 5) full-native ports + drop Electron.

**Decisions locked 2026-07-21:** **A** — standalone authenticated Node control-plane host for ACP + MCP + Companion, with the durable session broker remaining a separate process · **B** — notarized Developer ID / non-sandboxed direct distribution with deliberately secured per-user helpers · **C** — separate Electron/native stores with idempotent one-way import and an explicit write-owner at cutover · **D** — dual-broker drain, so existing Node-owned PTYs finish naturally while new sessions eventually move to the Swift daemon. The retired desktop Board is not a native parity target; its legacy Companion wire DTOs remain Codable for protocol/fixture compatibility but are never consumed by the native desktop UI.
